Live control of a physically plausible virtual biped, using the Kinect

Project description

This project has three main goals:

  • Live control of a virtual biped, using a video-based motion capture solution, specifically a Microsoft Kinect sensor.
  • Controlling a physically plausible virtual character, tracked using internal plausible joint torques.
  • Robust and balanced motion, including under limited external perturbation and interaction with the virtual environment.
Kinematic challenges include missing joints, noisy data and dynamic character proportions, imposed by using a Kinect sensor. Dynamics challenges include balance, maintaining global position and orientation and preventing foot sliding. Experiments will test live control, robustness with different types of motions and both passive and active interactions with the virtual environment.

This project was preluded by a small project, experimentally assessing the quality of Kinect-generated motion data, when used to drive a physics-based character. The report of this small project (with some personal details removed) can be found here (.pdf)

21-12-2012: Comparison of joint angles and joint positions from raw Kinect data

The video comparison below shows some of the kinematic challenges, when using a Kinect for motion tracking. When using absolute joint positions, the character remains relatively stable and its end-effectors are stable and accurate, when being correctly tracked. However, the distance between neighbouring joints varies from pose to pose. This means that joint positions, as returned by a Kinect, are not a sufficient kinematic target for a virtual character with fixed dimensions.

The Kinect for Windows SDK has functionality to convert those joint positions to both absolute and hierarchical joint angles. The character on the left in the video has fixed dimensions (bone lengths) and uses hierarchical joint angles. The global position of that character still uses absolute position data, but the joints are positioned relative to each other. As is apparent from the video, this method causes the character's end-effector positions to deviate significantly from the absolute positions seen on the right character. The left character floats or drops through the floor at certain poses.

The next course of action is to look into inverse kinematics techniques, to combine these two methods and get the best of both worlds, by tracking the absolute end-effector positions, while using the joint angles to drive the virtual character's pose towards that of the person standing in front of the Kinect.
